# Role: System / IT Administrator

The System Administrator manages the technical foundation, including tenants, identity, and integrations.

**Total Pages Accessible: 10**

---

## 1. Organisation Admin Dashboard
*   **Purpose:** Overview of platform health and setup.
*   **Content:** Tenant list, User counts, System alerts, Integration health.
*   **Primary Actions:** Create Tenant, View System Settings.

## 2. Tenant and Organisation Setup
*   **Purpose:** Configure the organizational hierarchy.
*   **Content:** Plant/Department structure, Node-specific settings.
*   **Primary Actions:** Add/Move/Rename Organisation Nodes.

## 3. User Directory & Invitations
*   **Purpose:** Manage platform access.
*   **Content:** Active users, Invitation status, Last login timestamps.
*   **Primary Actions:** Invite new user, Revoke access, Reset password.

## 4. Role and Permission Builder
*   **Purpose:** Define access control.
*   **Content:** Role list, Module-level permission matrix (Read/Write/Approve).
*   **Primary Actions:** Create custom role, Map permissions.

## 5. SSO Configuration
*   **Purpose:** Manage identity providers.
*   **Content:** SAML/OIDC settings, Certificate status.
*   **Primary Actions:** Enable SSO, Test connection.

## 6. Integration Monitoring & Jobs
*   **Purpose:** Track data flows with external systems (HR, ERP).
*   **Content:** Sync job history, Error logs, Failed record list.
*   **Primary Actions:** Retry failed job, Download error report.

## 7. Audit Log Viewer
*   **Purpose:** Forensic review of system activity.
*   **Content:** Event timeline (User, Action, Resource, Timestamp, Payload).
*   **Primary Actions:** Search logs, Export audit trail.

## 8. Data Quality Review
*   **Purpose:** Monitor for missing or invalid data.
*   **Content:** Invalid master data records, Missing mandatory fields.
*   **Primary Actions:** Resolve data issues.

## 9. System Settings
*   **Purpose:** Global platform configuration.
*   **Content:** Branding, Security policies, Notification global settings.

## 10. Help and Support
*   **Purpose:** Manage system documentation and support tickets.
